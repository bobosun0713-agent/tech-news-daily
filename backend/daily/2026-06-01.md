## 深入閱讀

### 1. [Shopify 以 BFS GraphQL 引擎把欄位執行速度提升最高 15 倍 (Shopify Reports 15X Faster Graphql Execution with Breadth First Engine)](https://www.infoq.com/news/2026/06/shopify-graphql-cardinal-bfs/?utm_campaign=infoq_content&utm_source=infoq&utm_medium=feed&utm_term=global)
* **核心重點：** Shopify 將 GraphQL 執行引擎改為 breadth-first traversal 與批次 resolver 處理後，在高 cardinality 查詢場景下取得最高 15 倍欄位執行速度、6 倍較低 GC 開銷與更好的延遲表現。
* **相關標籤：** `#GraphQL` `#Performance` `#Shopify`

### 2. [BadHost 漏洞可繞過 Host 驗證，波及 AI gateway 與代理系統 (BadHost Vulnerability Exposes AI Agents, Evaluators, and LLM Gateways)](https://www.infoq.com/news/2026/06/badhost-ai-systems-vulnerability/?utm_campaign=infoq_content&utm_source=infoq&utm_medium=feed&utm_term=global)
* **核心重點：** Starlette 出現高風險 BadHost 漏洞，攻擊者可利用異常 Host header 繞過部分路徑型存取控制；凡是建在其上的 AI gateway、agent 服務與 Web 後端都應儘速檢查與修補。
* **相關標籤：** `#Security` `#Starlette` `#AIGateway`

### 3. [AWS API Gateway 曾因尾斜線正規化差異繞過授權 (A Trailing Slash Bypassed AWS API Gateway Authorization)](https://www.infoq.com/news/2026/06/aws-api-gateway-auth-bypass/?utm_campaign=infoq_content&utm_source=infoq&utm_medium=feed&utm_term=global)
* **核心重點：** 研究人員揭露 AWS HTTP API 在路由比對與授權層間存在尾斜線正規化落差，導致某些 Lambda authorizer 規則可被繞過，提醒團隊重新審視 API gateway 的路徑授權假設。
* **相關標籤：** `#AWS` `#APIGateway` `#Auth`
